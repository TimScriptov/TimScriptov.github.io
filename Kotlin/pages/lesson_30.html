В языках программирования значение null - это отсутствие значения.

Если вы объявляете переменную в Kotlin, вы получаете значение, отличное от нуля, что означает, что переменная не может быть нулевой:

fun main(args: Array<String>) {
  var str: String = "Kotlin";
  str = null;
}

Этот код генерирует ошибку компиляции, потому что тип система Kotlin пытается исключить NullPointerException. Это гарантирует доступ к свойствам и методам str:

fun main(args: Array<String>) {
  var a: String = "Kotlin";
  println(a.length);// safely accessing a property
}

Чтобы разрешить null, мы можем объявить переменную как nullable, используя знак вопроса, как показано ниже:

fun main(args: Array<String>) {
  var str: String? = "Kotlin";
  str = null;
  println(str);
}

Приведенный выше код будет работать без ошибок.

Тем не менее, теперь, пытаясь получить доступ к свойству на nullable str, возникает ошибка времени компиляции, так как теперь str может иметь значение null:

fun main(args: Array<String>) {
  var str: String? = "Kotlin";
  println(str.length);
}

Чтобы получить безопасный доступ к значениям с нулевым значением, мы должны использовать оператор ?. (называемый оператором безопасного вызова).

fun main(args: Array<String>) {
  var str: String? = "Kotlin";
  println(str?.length);
}

? может использоваться с любым типом, чтобы сделать его нулевым.
